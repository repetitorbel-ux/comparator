# Руководство пользователя File Compare

## Обзор

`File Compare` это настольная утилита для Windows, предназначенная для сравнения файлов и каталогов.
Она умеет сравнивать два каталога, два явных файла или два набора выбранных элементов.

Типовые сценарии использования:

- сравнить содержимое левого и правого каталогов;
- увидеть, что есть только на одной стороне;
- найти несовпадения по размеру или дате изменения;
- просмотреть текстовые отличия для конкретной пары файлов;
- запускать сравнение из Total Commander как внешний инструмент.

Программа не синхронизирует каталоги и не копирует файлы автоматически.
Это инструмент сравнения, а не менеджер синхронизации.

## Поддерживаемые режимы

### 1. Сравнение каталогов

В этом режиме сравниваются все файлы из двух каталогов:

- левый каталог;
- правый каталог;
- при необходимости рекурсивный обход;
- при необходимости сравнение по размеру;
- при необходимости сравнение по дате изменения.

Это стандартный режим при запуске GUI напрямую или через:

```bash
file-compare-cli --left-dir "D:\Left" --right-dir "D:\Right" --size --date
```

### 2. Сравнение выбранных элементов

В этом режиме сравниваются только выбранные элементы с обеих сторон.

Это удобно, когда Total Commander передаёт не всё содержимое каталога, а только текущий selection.

Пример:

```bash
file-compare-cli --left-dir "D:\Left" --right-dir "D:\Right" --left-selected-list "D:\left.txt" --right-selected-list "D:\right.txt" --size --date
```

Важное ограничение:

- выбранные элементы должны быть переданы для обеих сторон либо не передаваться вообще.

### 3. Сравнение явной пары файлов

В этом режиме сравнивается один файл слева и один файл справа:

```bash
file-compare-cli --left-file "D:\Left\a.txt" --right-file "D:\Right\b.txt" --size --date
```

Только в этом режиме доступны встроенное редактирование и просмотр содержимого по строкам.

## Устройство GUI

Главное окно состоит из нескольких зон:

1. панель критериев сравнения;
2. переключатель языка;
3. кнопка запуска сравнения;
4. поля путей слева и справа для режима каталогов;
5. таблица результатов;
6. нижняя панель просмотра содержимого.

## Критерии сравнения

Панель критериев поддерживает:

- `Name`
- `Size`
- `Date Modified`
- `Include subdirectories (Recursive)`

Особенности поведения:

- в режимах каталогов и selected items сравнение имени всегда включено;
- в explicit file mode имена файлов могут отличаться, поэтому `Name` становится обычным переключаемым критерием.

## Состояния результата

Каждый элемент результата получает одно из состояний:

- `MATCH`
- `LEFT_ONLY`
- `RIGHT_ONLY`
- `MISMATCH`

Колонка `Details` показывает, по каким критериям найдено несовпадение, например:

- `Name`
- `Size`
- `Date`

## Панель diff содержимого

Когда выбрана подходящая пара файлов, нижняя панель показывает side-by-side diff:

- выровненные номера строк;
- визуальную подсветку изменённых строк;
- навигацию между отличиями;
- поля путей к левому и правому файлу.

Если результат существует только на одной стороне, противоположная сторона показывается как отсутствующая.

## Режим редактирования

`Edit Mode` доступен только в explicit file mode и только для UTF-8 текстовых файлов.

Когда редактирование доступно, можно:

- включить редактирование для обеих панелей;
- менять левый и правый текст независимо;
- сохранять только левую сторону;
- сохранять только правую сторону;
- повторно сравнивать после сохранения.

Кнопки этого режима:

- `Edit Mode`
- `Save Left`
- `Save Right`
- `Recompare`
- `Previous Difference`
- `Next Difference`

## Работа с несохранёнными изменениями

Если есть несохранённые изменения, программа спрашивает, что делать, перед:

- запуском нового сравнения;
- выполнением `Recompare`;
- закрытием окна.

Доступные варианты:

- сохранить изменения;
- отбросить изменения;
- отменить действие.

## Поддержка языков

GUI поддерживает:

- English;
- Russian.

Поведение локализации:

- выбранный язык хранится в `QSettings`;
- при следующем запуске используется сохранённое значение;
- если сохранённого значения нет, программа берёт язык из system locale.

## Аргументы командной строки

CLI launcher поддерживает:

```text
--left-file
--right-file
--left-dir
--right-dir
--recursive
--size
--date
--left-selected
--right-selected
--left-selected-list
--right-selected-list
```

Правила:

- `--left-file` и `--right-file` должны передаваться вместе;
- `--left-dir` и `--right-dir` должны передаваться вместе;
- selection mode требует selections и слева, и справа.
- если переданы `--left-dir` и `--right-dir`, режим file-pair не используется:
приложение выбирает selected-items (если есть selections с обеих сторон) или directories.

## Использование в Total Commander

Для Total Commander основные сценарии запуска такие.

### Универсальная кнопка (рекомендуется)

Можно использовать одну команду для всех типовых сценариев:

```text
--left-dir "%<Q" --right-dir "%>Q" --left-file "%P%N" --right-file "%T%M" --left-selected-list "%<$L" --right-selected-list "%>$L" --size --date
```

Приоритет выбора режима:

1. если есть selection lists с обеих сторон, используется режим выбранных элементов;
2. иначе используется сравнение каталогов.

Важно:
- если переданы `--left-dir` и `--right-dir`, параметры `--left-file` / `--right-file` игнорируются;
- чтобы принудительно запускать explicit file mode, используй отдельную команду без `--left-dir` / `--right-dir`.

### Сравнение текущих каталогов

```text
--left-dir "%<Q" --right-dir "%>Q" --size --date
```

### Сравнение текущей пары файлов

```text
--left-file "%P%N" --right-file "%T%M" --size --date
```

### Сравнение только выбранных элементов

```text
--left-dir "%<Q" --right-dir "%>Q" --left-selected-list "%<$L" --right-selected-list "%>$L" --size --date
```

## Установка и сборка

Установка зависимостей:

```bash
pip install -e ".[dev]"
```

Сборка Windows executable:

```bash
powershell -ExecutionPolicy Bypass -File .\scripts\build_exe.ps1
```

Ожидаемый результат сборки:

- `dist\FileCompareTC.exe`


## Ограничения

- редактирование поддерживается только в режиме явной пары файлов;
- редактирование поддерживается только для UTF-8 текстовых файлов;
- режим selected items симметричен и ожидает selections с обеих сторон;
- программа сравнивает файлы, но не синхронизирует и не копирует их;
- интеграция с Total Commander построена как внешний запуск процесса, а не как нативный TC plugin.
